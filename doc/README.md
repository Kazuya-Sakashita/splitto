# Splitto ドキュメント

このディレクトリは、
割り勘・立替精算アプリ **Splitto** の仕様・設計に関するドキュメントをまとめたものです。

実装前の合意形成、実装中の参照、後からの仕様確認を目的としています。

---

## 📘 ドキュメント構成

### 1. 概要
- [`overview.md`](./overview.md)

アプリ全体のコンセプト・目的・想定利用シーン・技術スタックをまとめています。
**初めて Splitto を見る人が最初に読むドキュメント**です。

---

### 2. 機能一覧
- [`features.md`](./features.md)

Splitto が提供する機能を網羅的に整理しています。

- 認証・アカウント
- グループ管理
- 支払い（立替）管理
- 割り方（Split）
- 清算
- メール連絡
- 通知
- 監査ログ

**「何ができるアプリか」を把握するための一覧**です。

---

### 3. 要件・仕様
- [`requirements.md`](./requirements.md)

ビジネスルール・権限・非機能要件など、
**実装時に必ず守るべき仕様**を定義しています。

- ロールと権限
- 割り方・清算ルール
- メール連絡の前提条件
- MVP / フェーズ分割

---

### 4. データモデル / ER 図
- [`er-diagram.md`](./er-diagram.md)

Splitto のデータ構造を **ER 図（Mermaid）＋テーブル説明**でまとめています。

- テーブル構成
- リレーション
- 主要カラムの役割
- 制約・ビジネスルール
- インデックス設計の考え方

**DB設計・API設計・集計ロジックの前提となる重要ドキュメント**です。

---

## 🧭 ドキュメントの読み方（おすすめ順）

1. **overview.md**
   → アプリの全体像を理解する

2. **features.md**
   → 実装すべき機能を把握する

3. **requirements.md**
   → 実装時のルール・制約を確認する

4. **er-diagram.md**
   → データ構造・リレーションを理解する
